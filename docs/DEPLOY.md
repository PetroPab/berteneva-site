# Развёртывание betneva-site.ru на Vercel

## Требования

- Аккаунт на [vercel.com](https://vercel.com)
- Аккаунт на [resend.com](https://resend.com) — для отправки писем с формы
- Домен `betneva-site.ru` (уже куплен)

---

## 1. Подключить репозиторий к Vercel

1. Зайти на [vercel.com/new](https://vercel.com/new)
2. Выбрать GitHub/GitLab/Bitbucket репозиторий с проектом
3. Framework Preset — **Next.js** (определится автоматически)
4. Root Directory — оставить пустым (проект в корне репо)
5. **Не нажимать Deploy** — сначала настроить переменные окружения (шаг 2)

---

## 2. Переменные окружения

В разделе **Environment Variables** добавить два значения:

| Ключ | Значение | Окружения |
|------|----------|-----------|
| `RESEND_API_KEY` | API-ключ из личного кабинета Resend | Production, Preview |
| `CONTACT_EMAIL` | Адрес, на который приходят заявки с сайта | Production, Preview |

### Как получить Resend API Key

1. Зарегистрироваться на [resend.com](https://resend.com)
2. Перейти в **API Keys → Create API Key**
3. Название — например `betneva-site`
4. Permission — **Sending access**
5. Скопировать ключ (показывается один раз)

### Как настроить домен в Resend

Письма отправляются с адреса `noreply@betneva-site.ru`. Для этого нужно верифицировать домен:

1. В Resend: **Domains → Add Domain** → вписать `betneva-site.ru`
2. Resend покажет три DNS-записи (TXT и CNAME)
3. Добавить их в DNS-панели регистратора домена
4. Нажать **Verify** — статус сменится на Verified (обычно до 30 минут)

---

## 3. Привязать домен

После первого деплоя:

1. В проекте на Vercel: **Settings → Domains**
2. Добавить `betneva-site.ru`
3. Vercel покажет DNS-записи для домена:
   - Если домен на регистраторе — добавить A-запись или CNAME
   - Если DNS управляется через Vercel — делегировать NS-серверы
4. Дождаться SSL-сертификата (автоматически, Let's Encrypt, до 5 минут)

---

## 4. Первый деплой

После настройки переменных нажать **Deploy**. Vercel:

1. Запускает `npm run build`
2. Собирает статические страницы (SSG) и API-маршруты (Edge/Node)
3. Публикует на CDN

Проверить результат: открыть Production URL из дашборда Vercel.

---

## 5. Последующие деплои

Каждый `git push` в ветку `main` запускает автодеплой. Ничего делать не нужно.

Для ручного передеплоя: **Vercel Dashboard → Deployments → Redeploy**.

---

## Локальная разработка

```powershell
# Обычный старт (если .next не удалялся)
npm run dev

# Чистый старт (всегда работает, удаляет кеш перед запуском)
npm run dev:clean
```

Если сайт отображается «без стилей» (как газета) — использовать `npm run dev:clean`.

Сервер запускается на [http://localhost:3000](http://localhost:3000).

### Переменные для локальной разработки

Файл `.env.local` (не коммитится в git):

```env
RESEND_API_KEY=re_ваш_ключ_здесь
CONTACT_EMAIL=ваш@email.ru
```

---

## Структура переменных окружения

```
.env.local          ← локально (в .gitignore, не коммитить)
Vercel Dashboard    ← для production и preview
```

Никогда не коммитить `.env.local` и не вставлять ключи в код.
