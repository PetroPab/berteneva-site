# Временный показ заказчику через поддомен

## Суть схемы

Next.js требует Node.js — на виртуальном хостинге его нет. Поэтому:

- Сайт деплоится на **Vercel** (бесплатно, 5 минут)
- В DNS-панели хостинга создаётся поддомен, который ведёт на Vercel
- Заказчик открывает `preview.ваш-домен.ru` — видит сайт

Форма обратной связи работает полностью, стилей не теряется.

---

## Шаг 1 — Задеплоить на Vercel

Следовать инструкции из `docs/DEPLOY.md`, шаги 1–4.

По итогу Vercel выдаст адрес вида `betneva-site-xxx.vercel.app` — это рабочий URL проекта.

> Переменные `RESEND_API_KEY` и `CONTACT_EMAIL` заполнить, чтобы форма работала. Если форма пока не нужна — можно пропустить, сайт откроется и без них.

---

## Шаг 2 — Добавить поддомен в Vercel

1. Открыть проект на Vercel → **Settings → Domains**
2. Нажать **Add Domain**
3. Ввести желаемый поддомен, например:
   ```
   preview.ваш-домен.ru
   ```
4. Vercel покажет инструкцию с DNS-записью — нужна строка **CNAME**:
   ```
   Тип:    CNAME
   Имя:    preview
   Значение: cname.vercel-dns.com
   ```
   Скопировать значение из интерфейса Vercel (оно всегда `cname.vercel-dns.com`).

---

## Шаг 3 — Добавить DNS-запись на хостинге

Зайти в панель управления своим хостингом (Timeweb, Beget, REG.RU или другой).

Найти раздел **DNS / Управление доменом / Зона DNS** для своего домена.

Добавить новую запись:

| Поле | Значение |
|------|----------|
| Тип | `CNAME` |
| Имя / Поддомен | `preview` |
| Значение / Цель | `cname.vercel-dns.com.` (точка в конце — норма) |
| TTL | `3600` (или оставить по умолчанию) |

Сохранить.

> Название поддомена (`preview`) — на ваш выбор. Можно использовать `betneva`, `demo`, `test` и т.п.

---

## Шаг 4 — Подождать и проверить

DNS обновляется от 5 минут до 2 часов (зависит от регистратора).

Как проверить, что запись применилась — в браузере или через онлайн-инструмент:

```
https://dnschecker.org/#CNAME/preview.ваш-домен.ru
```

Когда запись появится, Vercel автоматически выпустит SSL-сертификат. Сайт откроется по `https://preview.ваш-домен.ru`.

## Удалить после согласования

Когда сайт перейдёт на постоянный домен `betneva-site.ru`:

1. Vercel → **Settings → Domains** → удалить поддомен
2. В DNS-панели хостинга — удалить CNAME-запись `preview`
